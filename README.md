# ⛄️ 燐的代購系統

一套完全前端的代購管理工具，所有資料即時取自 Google 試算表，不需架設伺服器。

---

## 🚀 有什麼功能

| 功能 | 做什麼的 |
|------|----------|
| 🛒 購物車 | 選商品、選款式、自動算錢、一鍵複製明細 |
| 🔎 查訂單 | 輸入名字就能查自己的訂單狀態 |
| 🧮 二補計算機 | 自動算日本國內+國際運費 |
| 📦 團務進度 | 看各團到哪了（待下單/已下單/已抵台） |
| 📊 後台管理 | 有密碼才能看，統計所有買家訂單 |

---

## 📁 有哪些檔案

project/
├─ index.html          ← 主畫面
├─ utils.js            ← 共用小工具
├─ data.js             ← 讀取 Google 試算表資料
├─ cart.js             ← 購物車功能
├─ search.js           ← 買家查詢功能
├─ calc.js             ← 二補運費計算
├─ group-status.js     ← 團務進度表

└─ README.md           ← 你現在看的這個

所有 js 檔案要和 index.html 放同一個資料夾。

## 🛠️ 怎麼用

### 1. 準備 Google 試算表

你的試算表要有這 5 個分頁，名字不能錯：

| 分頁名稱 | 用途 |
|----------|------|
| 訂單 | 所有買家的訂單紀錄 |
| 團務 | 各團進度公告 |
| 運費計算 | 商品單價和運費規則 |
| 購物車 | 購物車顯示的商品清單 |
| 顧客填單 | 買家填的 Google 表單回覆 |

一定要把試算表「發佈到網路」！
檔案 → 共用 → 發佈到網路

### 2. 改試算表 ID

打開 utils.js，把最上面的 ID 換成你自己的：

const SPREADSHEET_ID = "你的試算表ID";

試算表 ID 就在網址 /d/ 和 /edit 中間那段。

### 3. 上傳檔案

所有檔案丟到 GitHub Pages 或 Netlify 或 Vercel 就可以了。
也可以直接放在資料夾裡，用瀏覽器打開 index.html。

## ⚠️ 常見問題

購物車沒東西？
→ 檢查試算表「購物車」分頁格式正不正確。

顯示連線失敗？
→ 試算表有沒有發佈到網路？ID 有沒有打錯？

怎麼加新團？
→ 在「購物車」分頁加團名和商品，在「團務」分頁加一筆團務資料。

---

## 📝 備註

所有資料都在你的試算表裡，這個網頁不會存任何東西。
二補運費是階梯式計算，規則寫在 data.js 裡面。
想改成員款式清單的話，去 utils.js 找 MEMBER_STYLES。
